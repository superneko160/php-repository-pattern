# Repositoryパターン

データアクセスのロジックとビジネスロジックを分離するデザインパターン。DDDで有名になった

## Usage

### Setup

1. import `setup.sql`
2. run composer command

```sh
composer install
composer dump-autoload
```

### Run

```sh
composer start
```

### Test

```sh
composer test
```

## 補足

このプロジェクトのディレクトリ構成はDDDっぽく作成しただけであり、完全にDDDの仕様に沿っているわけではない点に注意
